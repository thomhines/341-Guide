Positioning

Hey, you’re half way done! So now we are at the point where we should start moving things around, but how do you do that? I wish I could tell you drag and drop, but it’s not. When it comes to positioning items you have four essential tools. Your old friends' padding and margin (yup, there back!), and your new friends' position relative and position absolute. Each of these tools has their roles, advantages, and disadvantages when it comes to moving your content boxes around the web page.

Let’s start off easy with the stuff you already know. As discussed in the Box Model section, margin and padding adjust the position of the content within the area around it. This is useful for when you're trying to add space from the edge of an object(s). You can push objects away from each other by using fixed values or percentages.

Hot tip, when considering creating a responsive layout, the most efficient way to position elements is using margin and padding with percentage based values. When you use a percentage based values, it based on the parent container size which keeps things adjustable.

Position relative is probably the most confusing and misused positioning in CSS. What it means is "about itself." If you set position: relative; on an element but no other positioning attributes (top, left, bottom or right), it will have no effect on it's positioning at all. It will be exactly as it would be if you left it as position: static; But if you do give it some other positioning attribute, say, top: 10px;, it will shift it's position 10 pixels down from where it would normally be. In other words, relative positioning only visually moves the object, not physically within the flow of the website.

I'm sure you can imagine, the ability to shift an element around based on it's regular position is pretty useful. For example, you can use this to line up items that have a tendency not to want to line up.

Relative positioning is a very powerful type of positioning that allows you to place any page element exactly where you want it. You use the positioning attributes top, left, bottom and right to set the location. Remember that these values will be the next parent element with relative (or absolute) positioning. If there is no such parent, it will default all the way back up to the <html> element itself meaning it will be placed relative to the page itself.

The trade-off and most important thing to remember, about absolute positioning are that these items are removed from the flow of elements on the page. An object with this type of positioning is not affected by other items, and it doesn't affect other elements. A way to think of this process is like putting a sticker on a sheet of paper. The sticker does not fit within the words but is still a part of the composition.
