classes vs ID's

CSS sounds kind of cool right? It where the design becomes a reality in web design, and it can be a lot of fun too. So the first question is, how do you style elements? CSS relays on targeting elements, so it knows what you are trying to design, and this targeting process starts with using classes and ID’s.

First, to clarify, classes and ID’s serve the same purpose. Both tools are known as selectors, which gives elements a specified attribute that can be targetted in your CSS style sheet. Though they share a singular purpose, they operate under a different set of rules. Let’s see what those differences are.

Classes are “an identifier that can annotate multiple elements in a document.” In other words, you can use a class multiple times within document, to style various elements, under the same set of rules. Cool, right? Again it goes back to the idea of paragraph styles. Say if you have five items that you want to design the same way, you will use the same class title on all five of those <p> tags.

Adding a class to an element is pretty straightforward and fast and can be applied to nearly anything in your HTML document. All you have to do is add class=”______” to your element. So for example, adding a class to a <p> would look like this <p class=”class1”>...</p> note, I didn’t have to put the class style in the closing tag. When you add a class, you only have to do it in the beginning.

ID’s are “an identifier unique within the document,” hints the word unique. ID’s can only be used once in a single HTML document. If you try to use it a second time, Brackets (or whatever program you are using) will give you an error. For this reason, it’s common to see developers and designers utilizing classes vs. ID’s in most situations.

Now that you know what classes and ID’s are, how do you select them? Again, a simple process by using selectors.

Classes are defined by using a period and then placing the class name next to it. For example, if I wanted to select the class of “class1” in my CSS, I would do it by typing .class1{...}. ID’s work in a similar fashion but are rather defined by using the pound sign (#) instead of a period.

Note, selecting elements is case sensitive. If the case is not the same, your browser will be unable to match the class and the style together. This just leads to headaches and rough times.
